# Pulseer Dashboard

Pulseer Dashboard is a polished monitoring and alerting interface for Stellar and Soroban activity. It provides real-time visibility into watched accounts, contracts, events, and network health.

## Features

- **Activity Monitoring**: Track watched Stellar accounts and Soroban contracts.
- **Event Normalisation**: View unified event streams for transactions, operations, and contract events.
- **Alert Management**: Review and manage alerts generated by Pulseer rules.
- **Health Dashboard**: Monitor the status of blockchain infrastructure and sync lag.
- **Notification Previews**: Preview payloads ready for handoff to Slack, Discord, or webhooks.
- **Local Configuration**: Manage monitoring targets and rules directly in the UI.

## Role in the Ecosystem

Pulseer Dashboard serves as the primary interface for:
- **Protocol Maintainers**: Monitoring contract health and unexpected event topics.
- **Treasury Operators**: Tracking large payments and account activity.
- **Developers**: Debugging contract events and transaction success.
- **Indexer Operators**: Verifying sync status and ledger lag.

## Getting Started

### Local Setup

1.  **Clone the repository**.
2.  **Install dependencies**:
    ```bash
    npm install
    ```
3.  **Set up environment variables**:
    ```bash
    cp .env.example .env.local
    ```
4.  **Run the development server**:
    ```bash
    npm run dev
    ```
5.  Open [http://localhost:3000](http://localhost:3000) in your browser.

## Commands

- `npm run dev`: Start development server.
- `npm run build`: Build for production.
- `npm run start`: Start production server.
- `npm run lint`: Run ESLint.
- `npm run format`: Format code with Prettier.
- `npm run typecheck`: Run TypeScript type checking.
- `npm run test`: Run unit tests with Vitest.
- `npm run test:e2e`: Run end-to-end tests with Playwright.

## Security

Pulseer Dashboard is a **read-only** monitoring tool.
- It **never** requests or stores private keys.
- It **never** signs or submits transactions.
- All monitoring and alerts are advisory and require independent review.

Please see [SECURITY.md](./SECURITY.md) for more details.

## Limitations

- Monitoring coverage depends on the availability of configured Horizon and Soroban RPC endpoints.
- Alert delivery is advisory; this dashboard does not guarantee 100% coverage or real-time delivery in all network conditions.
- Status changes (acknowledgements, etc.) are stored locally unless a backend integration is configured.
